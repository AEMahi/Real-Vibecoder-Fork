## Quick Checklist

### Before Deploying:

- [ ] Read SETUP-GUIDE.md (full instructions)
- [ ] Deploy `cloudflare-worker.ts` as a Cloudflare Worker (using wrangler)
- [ ] Generate a secure secret and add to Cloudflare dashboard
- [ ] Add GitHub Secrets: CLOUDFLARE_WORKER_URL + CLOUDFLARE_EXPORT_SECRET
- [ ] Copy `.github-workflows-sync-logs.yml` → `.github/workflows/sync-logs.yml`
- [ ] Add `scripts/sync-prompt-logs.js` to your repo
- [ ] Add `scripts/analyze-and-improve-filter.ts` to your repo
- [ ] Copy `wrangler.toml` to root (update KV namespace IDs after first wrangler deploy)
- [ ] Update `src/promptLog.ts` with the new analytics-tracking version
- [ ] Update `src/routes/p/$projectId.tsx`:
  - Import trackVisit: `import { trackVisit } from "./promptLog"`
  - Call it on mount: `useEffect(() => trackVisit(), [])`
  - Set env vars: `VITE_PROMPT_LOG_URL` + `VITE_ANALYTICS_URL`

### After Deploying:

- [ ] Test the "Use Prompts to Improve" button (should just flip a toggle, no picker)
- [ ] Send a test prompt while logging is ON
- [ ] Wait ~6 hours or manually trigger GitHub Actions workflow
- [ ] Check `logs/prompt-filter-log.json` in your repo (should have entries)
- [ ] Run `bun run scripts/analyze-and-improve-filter.ts` to see stats
- [ ] Review filter performance and update rules as needed

## File Purposes (Quick Reference)

| File | Purpose | Who Owns It |
|------|---------|-----------|
| `promptFilter.ts` | Filter rules (run client-side) | You maintain |
| `promptLog.ts` | Send logs to Cloudflare (client) | Auto-generated, don't edit |
| `cloudflare-worker.ts` | Receive logs, store in KV, export | Deployed separately |
| `wrangler.toml` | Cloudflare Worker config | You customize with IDs |
| `sync-logs.yml` | GitHub Actions workflow | Auto-runs every 6h |
| `sync-prompt-logs.js` | Pulls logs from Cloudflare | Runs via GitHub Actions |
| `analyze-and-improve-filter.ts` | Analyzes logs for patterns | You run manually |
| `logs/prompt-filter-log.json` | Committed log archive | Auto-generated by GitHub Actions |

## Environment Variables to Set

### In `.env.production` (frontend build):
```
VITE_PROMPT_LOG_URL=https://blanksheet-worker.<subdomain>.workers.dev/api/prompt-log
VITE_ANALYTICS_URL=https://blanksheet-worker.<subdomain>.workers.dev/api/analytics
```

### In GitHub Secrets (for Actions):
```
CLOUDFLARE_WORKER_URL=https://blanksheet-worker.<subdomain>.workers.dev
CLOUDFLARE_EXPORT_SECRET=<random 32-char hex string>
```

### In Cloudflare Worker Dashboard (Production):
```
EXPORT_SECRET=<same random string as above>
```

## Testing Before Going Live

1. Deploy Worker to a test subdomain first
2. Update frontend to point to test URL
3. Send a few prompts with logging ON
4. Manually call the export endpoint:
   ```bash
   curl -H "Authorization: Bearer <your-secret>" \
        https://test-worker.workers.dev/api/prompt-log/export
   ```
5. Verify logs appear and are properly formatted JSON
6. Then flip to production URLs and GitHub Secrets

## Support

If something breaks:

1. Check Cloudflare Dashboard → Workers → Logs (see actual errors)
2. Check GitHub Actions → sync-logs workflow (see if sync failed)
3. Check browser DevTools → Network → api-prompt-log POST (see if request succeeded)
4. Fallback: All filtering still works client-side even if logging fails
